# 数据库设计（优化版）

## 设计理念

基于微信小程序云开发数据库（文档型数据库）的特性，采用面向文档的设计理念，充分利用嵌套文档和数组结构，减少关联查询，提高查询效率。

## 集合设计

### 1. users集合（用户信息）

```javascript
{
  _id: "user123",              // 用户ID（自动生成）
  nickname: "张三",            // 用户昵称
  avatar: "cloud://xxx/avatar.png",  // 用户头像（云存储路径）
  familyId: "family456",       // 所属家庭ID
  createdAt: "2024-01-01T12:00:00Z"  // 创建时间
}
```

**字段说明：**
- `_id`: 系统自动生成的唯一标识
- `nickname`: 用户昵称，用于显示
- `avatar`: 用户头像，存储在云存储中的图片路径
- `familyId`: 关联families集合的家庭ID
- `createdAt`: 用户创建时间戳

### 2. families集合（家庭信息）

```javascript
{
  _id: "family456",            // 家庭ID（自动生成）
  name: "幸福家庭",           // 家庭名称
  members: [                   // 家庭成员列表
    { 
      userId: "user123", 
      role: "管理员", 
      joinedAt: "2024-01-01T12:00:00Z" 
    },
    { 
      userId: "user789", 
      role: "成员", 
      joinedAt: "2024-01-02T12:00:00Z" 
    }
  ],
  createdAt: "2024-01-01T12:00:00Z"  // 创建时间
}
```

**字段说明：**
- `_id`: 系统自动生成的唯一标识
- `name`: 家庭名称
- `members`: 家庭成员数组，包含每个成员的用户ID、角色和加入时间
- `createdAt`: 家庭创建时间戳

### 3. recipes集合（食谱信息）

```javascript
{
  _id: "recipe123",            // 食谱ID（自动生成）
  name: "番茄炒蛋",           // 食谱名称
  image: "cloud://xxx/tomato-egg.png",  // 食谱图片（云存储路径）
  notes: "1. 准备食材...",     // 用户自选备注，可以是制作步骤也可以是其他信息
  ingredients: [               // 所需食材列表
    { 
      ingredientId: "ing456", 
      name: "番茄", 
      quantity: 2, 
      unit: "个" 
    },
    { 
      ingredientId: "ing789", 
      name: "鸡蛋", 
      quantity: 3, 
      unit: "个" 
    }
  ],
  familyId: "family456",       // 所属家庭ID
  creatorId: "user123",        // 创建者用户ID
  createdAt: "2024-01-01T12:00:00Z"  // 创建时间
}
```

**字段说明：**
- `_id`: 系统自动生成的唯一标识
- `name`: 食谱名称
- `image`: 食谱图片，存储在云存储中的图片路径
- `steps`: 烹饪步骤描述
- `ingredients`: 所需食材数组，包含食材ID、名称、数量和单位
- `familyId`: 关联families集合的家庭ID
- `creatorId`: 关联users集合的创建者ID
- `createdAt`: 食谱创建时间戳

### 4. ingredients集合（食材信息）

```javascript
{
  _id: "ing456",               // 食材ID（自动生成）
  name: "番茄",               // 食材名称
  quantity: 5,                // 剩余数量
  unit: "个",                 // 数量单位（个/克/千克等）
  type: "蔬菜",               // 食材类型（蔬菜/肉类/调料等）
  expiryDate: "2024-01-10T23:59:59Z",  // 过期时间
  uploaderId: "user123",      // 上传者用户ID
  familyId: "family456",       // 所属家庭ID
  createdAt: "2024-01-01T12:00:00Z"  // 添加时间
}
```

**字段说明：**
- `_id`: 系统自动生成的唯一标识
- `name`: 食材名称
- `quantity`: 剩余数量
- `unit`: 数量单位（个、克、千克、包等）
- `type`: 食材类型（蔬菜、肉类、调料、主食等）
- `expiryDate`: 过期时间，用于提醒用户食材即将过期
- `uploaderId`: 关联users集合的上传者ID
- `familyId`: 关联families集合的家庭ID
- `createdAt`: 食材添加时间戳

## 设计优势

### 1. 查询效率提升
- 通过嵌套结构减少跨集合查询
- 一次性获取相关数据，减少网络请求次数

### 2. 数据完整性保证
- 相关数据存储在一起，避免数据不一致
- 减少了外键关联的复杂性

### 3. 灵活性和可扩展性
- 文档结构支持灵活扩展，无需修改表结构
- 支持复杂的数据模型和业务场景

### 4. 业务需求支持
- 支持同一家庭多用户共享食材和食谱
- 支持根据食材剩余量推荐食谱的功能
- 支持食材过期提醒功能

```javascript
db.collection('ingredients').where({
  familyId: 'family456'
}).get()
```

## 注意事项

1. **索引优化**：为常用查询字段创建索引，提高查询性能
2. **权限控制**：设置适当的数据库权限规则，确保数据安全
3. **数据验证**：在应用层进行数据验证，确保数据质量
4. **定期清理**：定期清理过期数据，保持数据库性能
