# 云函数部署与测试指南

## 一、云函数部署步骤

### 1. 准备工作

#### 1.1 检查微信开发者工具配置
- 打开微信开发者工具
- 确保云开发环境已正确配置
- 确认环境ID已设置

#### 1.2 检查云函数配置
- 云函数目录：`cloudfunctions/nonecookService/`
- 依赖包：`wx-server-sdk ~2.6.3`
- 主文件：`index.js`

### 2. 部署方法

#### 方法一：使用微信开发者工具部署（推荐）

1. **右键部署**
   - 在微信开发者工具中，右键点击`nonecookService`云函数
   - 选择"上传并部署：云端安装依赖（不上传node_modules）"

2. **等待部署完成**
   - 部署过程会自动安装依赖包
   - 部署完成后会显示"部署成功"提示

#### 方法二：使用命令行部署

使用项目中的上传脚本：
```bash
# 修改脚本中的参数
# envId: 你的云开发环境ID
# projectPath: 项目路径

# 执行部署
bash uploadCloudFunction.sh
```

### 3. 部署注意事项

- 确保网络连接稳定
- 云函数代码不要超过50MB
- 首次部署可能需要较长时间
- 部署完成后，云函数会自动在云端运行

## 二、云函数测试方法

### 1. 使用微信开发者工具测试

#### 1.1 在云开发控制台测试

1. **打开云开发控制台**
   - 在微信开发者工具中点击"云开发"按钮
   - 进入云开发控制台

2. **测试云函数**
   - 选择"云函数" -> "nonecookService"
   - 点击"测试"按钮
   - 输入测试参数：
     ```json
     {
       "module": "user",
       "action": "ensureUserHasFamily",
       "userId": "user123"
     }
     ```
   - 点击"运行"查看结果

#### 1.2 在小程序中测试

1. **启动小程序**
   - 在微信开发者工具中编译并预览小程序

2. **测试各个功能**
   - 点击个人中心，测试用户信息加载
   - 点击"创建家庭"，测试家庭创建功能
   - 点击"加入家庭"，测试家庭加入功能
   - 在冰箱页面测试食材管理功能
   - 在菜谱页面测试菜谱管理功能

### 2. 使用日志查看测试结果

#### 2.1 查看云函数日志

1. **打开云开发控制台**
   - 选择"云函数" -> "nonecookService"
   - 点击"日志"标签

2. **查看详细日志**
   - 可以查看每次调用的详细日志
   - 包括调用参数、执行时间、返回结果
   - 错误信息会显示在日志中

#### 2.2 查看小程序控制台日志

1. **打开小程序控制台**
   - 在微信开发者工具中点击"控制台"

2. **查看前端日志**
   - 查看API调用结果
   - 查看错误信息和警告
   - 查看数据加载状态

## 四、常见问题及解决方案

### 1. 部署失败

**问题**：部署时显示"依赖安装失败"
**解决方案**：
- 检查网络连接
- 确认wx-server-sdk版本正确
- 尝试手动安装依赖后再部署

### 2. 调用失败

**问题**：调用云函数返回错误
**解决方案**：
- 检查云函数日志，查看具体错误信息
- 确认参数格式正确
- 检查权限设置

### 3. 权限问题

**问题**：调用云函数显示"权限不足"
**解决方案**：
- 在云开发控制台设置数据库权限
- 确保用户已登录
- 检查云函数代码中的权限验证

### 4. 数据同步问题

**问题**：前端显示的数据与数据库不一致
**解决方案**：
- 检查数据刷新机制
- 确认云函数返回的数据格式正确
- 检查前端数据绑定逻辑

## 五、测试流程

### 1. 初始化测试

1. **初始化数据库**
   - 调用init模块的云函数
   - 确认数据库集合创建成功

2. **测试用户初始化**
   - 调用ensureUserHasFamily函数
   - 确认用户有专属家庭

### 2. 功能测试

1. **按模块测试**
   - 用户模块测试
   - 家庭模块测试
   - 食材模块测试
   - 菜谱模块测试

2. **按场景测试**
   - 创建家庭场景
   - 加入家庭场景
   - 食材管理场景
   - 菜谱管理场景

### 3. 性能测试

1. **响应时间测试**
   - 测试云函数平均响应时间
   - 确保响应时间在合理范围内（<1秒）

2. **并发测试**
   - 测试多用户同时访问的稳定性
   - 确保系统不会崩溃

## 六、部署验证

### 1. 部署后验证

1. **检查云函数状态**
   - 在云开发控制台查看云函数状态
   - 确认云函数已正常运行

2. **验证数据库**
   - 检查数据库集合是否创建
   - 确认初始数据是否正确

3. **验证权限设置**
   - 测试不同用户权限
   - 确保数据访问安全

### 2. 线上测试

1. **真机测试**
   - 在真实设备上测试
   - 确认所有功能正常

2. **多端测试**
   - 在不同设备上测试
   - 确保兼容性

## 七、维护建议

### 1. 定期检查

- 定期查看云函数日志
- 监控云函数运行状态
- 及时修复发现的问题

### 2. 性能优化

- 优化数据库查询
- 减少不必要的API调用
- 缓存常用数据

### 3. 安全维护

- 定期更新依赖包
- 检查权限设置
- 防止SQL注入和其他安全问题
